Class and ID and Selectors

Selectors come in 3 forms: HTML Tag, class, and ID. Class and ID allow you to define your own selectors. This allows you to have the same HTML element presented differently from its counterparts.
In CSS the class selector is preceded by a full stop "." then the class name. While an ID selector is preceded by a "#".

In CSS you can also apply a selector to a specific HTML element by doing element.class or element.id.

For example p.list will only be applied to paragraph elements within the list class.

Grouping and Nesting

Two ways to simplify HTML and CSS while also making it easier to manage

Grouping allows for you to combine multiple selectors under one group and give each selector the attributes within it.

Nesting allows you to specify properties to selectors within other selectors.

Pseudo Classes

Purpose: to specify a state or relation to the selector it is attached to.
Most basic use is on a link to say if it was visited yet or not.

Dynamic Pseudo Class

Can be used to apply styles when something happens to something. Also commonly used for links.
Three main classes:
  1. active - used when something is activated by the user, such as a link being clicked on
  2. hover - used when something is passed over by an input from the user, such as a cursor moving over a link
  3. focus - used when something gains focus, when it is ready for or selected by for keyboard input
    • Most often used for form elements.
Another class is the first-child class. Its purpose is to target something only if it is the very first descendant of an element.
There are also last-child, target, and first-of-type classes to name a few more.

Shorthand Properties

Some CSS properties allow for a string of values to make it quicker to write.

Margins and Padding

Margins and padding can be written in the form of property: top right bottom left instead of writing each individually.
Same goes for padding as they can be written as property: (top and bottom) (right and left).

Borders and Fonts

The same can be done with borders and fonts. Just write it as one line instead of multiple like shown in the CSS file.
Border style, width, and color can be written as one by doing border: width color style. While font style, weight, size, line height, and family can be rewritten as font: style weight size line-height family.

Background Images

background: is an additional shorthand property.
It contains the properties of background: color image repeat position:
  • background-color: color of the background
  • background-image: location of the image
  • background-repeat: Can be one of 4 things.
    1. repeat - tile effect across whole background
    2. repeat-y - repeating on the y-axis, top and bottom
    3. repeat-x - repeating on the x-axis, left and right
    4. no-repeat - shows image just once
  • background-position: can be top, center,bottom,left,right, a length, a percentage, or any sensible combination such as top right
There are also other features that it can hold such as attachment, clip, origin, and size. See the property reference on google to see all of them.

Specificity

For when you have two or more conflicting CSS rules that point to the same element.

There are some basic rules that the browser will follow to figure out which one will win out.

  • More Specific = Greater Precedence
    • If the selectors are the same the last one will be used.
  • Calculating Specificity
    • ID Selector (#something) is a value of 100
    • Class Selector (.something) is a value of 10
    • HTML Selector (something) is a value of 1
    • Add these up and you get a specific value for that peice of code.
    • The higher the number the more specific it is and therefore the more often it will win out.
