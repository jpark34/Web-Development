Class and ID and Selectors

Selectors come in 3 forms: HTML Tag, class, and ID. Class and ID allow you to define your own selectors. This allows you to have the same HTML element presented differently from its counterparts.
In CSS the class selector is preceded by a full stop "." then the class name. While an ID selector is preceded by a "#".

In CSS you can also apply a selector to a specific HTML element by doing element.class or element.id.

For example p.list will only be applied to paragraph elements within the list class.

Grouping and Nesting

Two ways to simplify HTML and CSS while also making it easier to manage

Grouping allows for you to combine multiple selectors under one group and give each selector the attributes within it.

Nesting allows you to specify properties to selectors within other selectors.

Pseudo Classes

Purpose: to specify a state or relation to the selector it is attached to.
Most basic use is on a link to say if it was visited yet or not.

Dynamic Pseudo Class

Can be used to apply styles when something happens to something. Also commonly used for links.
Three main classes:
  1. active - used when something is activated by the user, such as a link being clicked on
  2. hover - used when something is passed over by an input from the user, such as a cursor moving over a link
  3. focus - used when something gains focus, when it is ready for or selected by for keyboard input
    • Most often used for form elements.
Another class is the first-child class. Its purpose is to target something only if it is the very first descendant of an element.
There are also last-child, target, and first-of-type classes to name a few more.
